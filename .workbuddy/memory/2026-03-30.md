## Shapefile 上传失败问题

### 问题
用户在 Shapefile 管理页面上传 ZIP 文件时显示"上传失败"，服务器无响应日志。

### 排查过程
1. 检查 PM2 日志 - 未发现新的 Shapefile 上传请求日志
2. 检查 nginx 配置 - 发现 API 代理配置正确
3. 检查前端代码 - 发现 ShapefileView.vue 使用了错误的 API 地址

### 根本原因
ShapefileView.vue 第69行：
```javascript
const baseURL = import.meta.env.VITE_API_BASE_URL || 'http://localhost:3000'
```
生产环境下使用 `localhost:3000` 导致请求发送到用户本地而非服务器。

### 修复
改为空字符串（相对路径）：
```javascript
const baseURL = import.meta.env.VITE_API_BASE_URL || ''
```

### 部署
1. 执行 `vite build` 重新构建前端
2. 用 paramiko/SFTP 上传 dist 目录到服务器
3. nginx -s reload 重载配置

### 额外修复
- 删除了 nginx sites-enabled 中的备份配置文件，解决 "conflicting server name" 警告

## Shapefile Polygon 坐标解析严重 Bug 修复 (16:42)

### 问题
点击显示按钮后浏览器白屏/死机

### 根本原因
shapefile_parser.py 中 Polygon 坐标解析逻辑错误：
- 原代码对每个 part 都从起始索引遍历到 `len(shape.points)`（最后一个点）
- 这会导致坐标嵌套错误，生成无效的 GeoJSON
- Leaflet 解析无效坐标时会导致浏览器崩溃

### 修复
正确的坐标解析逻辑：
```python
parts = list(shape.parts) + [len(shape.points)]  # 添加最后一个边界
for i, part_start in enumerate(shape.parts):
    part_end = parts[i + 1]  # 下一个 part 的起始索引
    # 正确遍历 part 范围内的点
```

- MapView.vue 中添加 try-catch 和 GeoJSON 格式验证
- 更详细的错误提示

## Shapefile 检索白屏问题修复（第9次）

### 问题
点击"在地图上显示"后页面立即白屏（白屏死机）

### 根本原因
1. `initMap()` 是异步函数但未被 await，导致在 1500ms 后检查 `window.shapefileQueryResult` 时 map 可能未初始化
2. 255 个要素同时处理可能导致浏览器阻塞

### 修复内容
**MapView.vue**:
1. 添加 `shapefileProcessing` 标志防止重复处理
2. 使用 `await initMap()` 确保地图初始化完成后再处理数据
3. 将 255 个要素分批处理（每批 20 个），使用 `requestAnimationFrame` + `setTimeout` 让浏览器喘口气
4. 使用 Polyline 只画边界线（不使用 Polygon 填充）
5. 减少延迟时间到 500ms（因为 initMap 已 await）

已部署到服务器 (19:35)，请用户测试。
