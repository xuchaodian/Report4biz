# 2026-03-29 工作日志

## 修复：我的门店导入CSV时store_type默认值错误
- **问题**：导入CSV时门店类型（store_type）字段为空，默认显示"社区店"（该值不在下拉列表中）
- **修复**：`backend/src/routes/markers.js` 中两处默认值从 `'社区店'` 改为 `'已开业'`
  - POST /api/markers（手动创建）
  - POST /api/markers/import（CSV导入）
- **规则**：CSV中store_type有值时以CSV为准，为空时默认"已开业"

## 新增：地图与数据管理页面筛选联动
- **机制**：三个 Pinia store（marker/competitor/brandStore）各新增 `visibleIds` 状态（null=显示全部，数组=仅显示这些ID）和 `setVisibleIds(ids)` action
- **MapView**：监听各 store 的 `visibleIds`，变化时调用 `reload*Layer()` 重绘对应图层
- **DataView/CompetitorView/BrandStoreView**：筛选变化时调用 `syncVisibleIds()` 同步过滤ID；全清除/批量删除后重置 visibleIds=null，恢复显示全部
- 全清除/批量删除后也会清空所有筛选条件

## 新增：三个门店页面全清除功能
- **涉及页面**：我的门店（DataView）、竞品门店（CompetitorView）、品牌门店（BrandStoreView，仅管理员可见）
- **后端**：三个路由各新增 `DELETE /clear-all` 接口
  - markers：清除当前用户自己的数据
  - competitors：普通用户清自己的，管理员清所有
  - brand-stores：仅管理员，清除全部
- **前端**：三个 store 各加 `clearAll*` action；三个 View 各加全清除按钮（带二次确认弹窗）
- 已构建并部署服务器，PM2 已重启
